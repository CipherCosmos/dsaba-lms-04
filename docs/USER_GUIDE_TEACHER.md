# Teacher User Guide - DSABA LMS

## Welcome to DSABA LMS

As a teacher, the DSABA Learning Management System (LMS) provides you with powerful tools to manage student marks, track academic performance, and generate comprehensive reports. This guide covers all features available to teaching faculty.

## Getting Started

### Logging In

1. Navigate to your institution's LMS URL
2. Enter your **username** and **password**
3. Click **Login**

**Security Note**: Always log out when finished, especially on shared computers.

### Teacher Dashboard

Your dashboard provides:

- **Class Overview**: Students and subjects you're teaching
- **Pending Tasks**: Marks to be entered, approvals needed
- **Recent Activity**: Latest updates in your classes
- **Quick Actions**: Fast access to common tasks

## Managing Student Marks

### Entering Internal Assessment Marks

1. Navigate to **Marks** > **Internal Marks**
2. Select your subject and batch
3. Choose the assessment type (IA1, IA2, Assignment, etc.)
4. Enter marks for each student:
   - Use the table view for bulk entry
   - Or individual entry for detailed input
5. Save and validate marks
6. Submit for approval if required

### Bulk Mark Entry

1. Go to **Marks** > **Bulk Upload**
2. Download the template (Excel/CSV)
3. Fill in student marks
4. Upload the file
5. Review and confirm import

**Tip**: Templates include validation rules to prevent errors.

### Final Mark Computation

1. Access **Marks** > **Final Marks**
2. Select subject and batch
3. The system automatically calculates:
   - Best internal marks (highest of IA1/IA2)
   - Total marks (internal + external)
   - Grade and percentage
4. Review calculations
5. Publish marks when ready

### Mark Approval Workflow

1. **Draft**: Initial entry, can be edited
2. **Submitted**: Sent for approval, read-only
3. **Approved**: Verified by HOD/coordinator
4. **Published**: Visible to students

## CO-PO Framework

### Understanding COs and POs

- **Course Outcomes (COs)**: Learning objectives for your subject
- **Program Outcomes (POs)**: Program-wide learning goals
- **Attainment**: How well students are meeting these outcomes

### Setting Up COs

1. Navigate to **CO-PO** > **Course Outcomes**
2. For each subject, define:
   - CO code (CO1, CO2, etc.)
   - Description of learning objective
   - Bloom's taxonomy level
   - Weightage percentage

### Mapping COs to POs

1. Go to **CO-PO** > **Mappings**
2. For each CO, map to relevant POs
3. Set correlation levels:
   - **High**: 3 (strongly related)
   - **Medium**: 2 (moderately related)
   - **Low**: 1 (weakly related)

### Attainment Calculation

1. Access **CO-PO** > **Attainment**
2. View attainment levels based on:
   - Student performance in assessments
   - CO-PO mapping correlations
   - Threshold settings

## Frequently Asked Questions

### Q: How do I correct entered marks?
**A**: Use the edit function before submitting for approval. After approval, contact your HOD for corrections.

### Q: Can I see marks entered by other teachers?
**A**: Only administrators and HODs can view marks across subjects. Teachers see only their subjects.

### Q: How do I handle absent students?
**A**: Mark them as absent in the attendance system. Their marks can be entered later when available.

### Q: What if a student disputes a mark?
**A**: Document the dispute and follow your institution's revaluation procedures.

### Q: How do I export data for external analysis?
**A**: Use the Export function in Reports section. Multiple formats are available.
